Output Types

When creating a formula field, the user selects one of 8 output types. This determines how the computed value is displayed and which presets are shown.

# Output type Description Example value Notes
1 Date only A single date with no time component Jun 30
2 Date range A start date + end date pair Jun 30 – Jul 14 Formula computes the end date; start is configured separately
3 Date time only A single date with time component Jun 30, 09:00
4 Date time range A start datetime + end datetime pair Jun 30 09:00 – Jul 14 17:00 Formula computes end datetime; start is configured separately
5 Time only Time of day, no date 09:00 Requires DATEADD / DATEDIF with "hours" or "minutes" units
6 Time range A start time + end time pair 09:00 – 17:00 Formula computes end time; start is configured separately
7 Number A numeric value 1,250
8 Number with currency Numeric with currency symbol and formatting ¥1,250 / $12.50 / €1,250 Currency (JPY, USD, BRL, KRW, EUR, DON) and comma-separator are configured separately; formula logic is identical to Number

Output Type Change Warnings

When a user changes the output type of a formula field that already has a formula, the system checks for compatibility. There are 6 warning patterns. The full FROM→TO matrix follows.

ℹ️
Non-destructive first. Always preserve the formula when possible. Only require the user to clear and rewrite when the types are fundamentally incompatible (Pattern A). For all other patterns, show a contextual warning and allow the user to proceed.

Warning patterns

Pattern A Fully incompatible
Formula returns the wrong fundamental type (e.g. date formula → number output). The user must clear or rewrite the formula before the type change takes effect. UI action: block the type change with a confirmation dialog offering "Clear formula and switch" or "Cancel".
Pattern B Single → Range
The formula currently computes a single value, but the new output type requires two endpoints (start + end). The formula will apply to the end value; the user must configure the start separately. Show a soft warning and allow the user to proceed.
Pattern C Range → Single
The formula was computing one endpoint of a range. The other endpoint (start) was configured outside the formula and will now be ignored. Show a soft warning: "The configured start value will no longer apply."
Pattern D Time stripped
Formula returns a datetime value, but the new output type is date-only or date-range. The time component will be dropped from the output. Show a soft warning and allow the user to proceed.
Pattern E Time defaulted
Formula returns a date-only value, but the new output type includes a time component. The time will default to 00:00. Show a soft warning and allow the user to proceed.
Pattern F Context-dependent
Compatibility depends on which functions and fields the formula uses. For example, DATEDIF returns a number, while DATEADD returns a date. Show a soft warning asking the user to verify the formula returns the expected type, and allow the user to proceed.
